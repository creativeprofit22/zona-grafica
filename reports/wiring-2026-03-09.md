## Detailed Traces

### 1. CMS Content Blocks — Incomplete Feature (BACKLOG)

Admin side fully built, public side not started. Expected for a demo.

| Layer | Status |
|-------|--------|
| DB table, read/write functions, API route | Working |
| Admin list + editor UI | Working |
| **Public pages reading CMS** | Not wired — pages import `data/*.ts` |

No content adapter exists. When ready to go live, either replace `data/*.ts` imports with `getContent()` calls or build a fallback adapter.

### 2. Contact + Newsletter — Functional Demo (BACKLOG)

Both forms work end-to-end (client → API → DB). The data lands correctly. What's missing:
- Admin pages to view submissions/subscribers
- Email notifications on new contact
- Newsletter unsubscribe flow

These are go-live requirements, not demo requirements.

### 3. Blog — `featured` Field Dropped (MEDIUM)

This is a real bug even in demo mode:

```typescript
// lib/blog.ts — BlogPostMeta interface missing featured
export interface BlogPostMeta {
  slug: string; title: string; excerpt: string; category: string;
  date: string; isoDate: string; readingTime: string;
  gradientFrom: string; gradientTo: string; image?: string;
  // featured?: boolean;  ← MISSING
}
```

`PostGrid.tsx:59` destructures `const [featured, ...rest] = posts` — uses chronological order, not the `featured` flag from frontmatter. With only 2 posts this is invisible, but will produce wrong results as posts are added.

### 4. Blog Article JSON-LD Missing Image (LOW)

One-line fix in `app/(site)/blog/[slug]/page.tsx:44-50` — add `image: post.meta.image` to the `articleSchema()` call. Both blog posts define images in frontmatter.

### 5. Rate Limiting — Clean

All 4 routes correctly wired. One minor inconsistency: contact route returns Spanish error, others return English.

### 6. Auth Flow — Clean

Session cookie `dd_admin_session` consistent everywhere. Admin layout gate + API content route both check `verifySession()`. One edge case: `login()` doesn't catch `getSigningSecret()` throw (dev-only, missing env var scenario).

### 7. CSP Nonce — Clean

Full pipeline works: `proxy.ts` → `x-nonce` header → `app/layout.tsx` → `<body nonce>`.

### 8. Analytics Pipeline — Clean

Perfect field consistency across all layers. All collected data is queried and displayed in admin dashboard. Country field is Vercel-only (handled gracefully).

### 9. Data Files — Clean

All 14 exports consumed. Zero dead exports. Zero type mismatches.

## Systemic Pattern: "Backend Done, Frontend Pending"

CMS, contact admin, and newsletter admin all share the same shape — data flows into the database but admin UI to read it back isn't built yet. This is fine for a demo, but all three need to be wired before go-live.
