## Systemic Patterns

### Pattern 1: Data Layer Bypass (3 instances)
The `data/` directory was designed as a centralized content source, but components frequently hardcode the same content internally:
- **HomeData hero/manifesto** → HeroSection + ManifestoSection hardcode text
- **clients.ts** → ClientMarquee hardcodes its own list
- **blog.ts** → never imported at all

This creates an illusion of a single source of truth while the actual truth lives scattered in components.

### Pattern 2: Built-But-Not-Wired Features (4 instances)
Full backend implementations exist for features that have no frontend consumer:
- **CMS content blocks** — full CRUD in admin, zero reads on site
- **Newsletter API** — full endpoint, no form
- **Admin charts** — real DB queries, stub rendering
- **Blog utilities** — `getFeaturedPosts()`, `getRecentPosts()`, `getCategoryMeta()` — all unused

### Pattern 3: Previous-Project Residue
The admin panel was ported from "Douro Digital" with incomplete adaptation:
- Brand name in 4 locations
- Content key descriptions reference Douro's content model
- Content keys don't map to Zona Gráfica's data structure
