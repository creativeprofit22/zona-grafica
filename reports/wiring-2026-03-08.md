## Findings

### CRITICAL — Silent Functional Breakage

#### 1. Newsletter Endpoint Completely Orphaned
- **WHERE**: `app/api/newsletter/route.ts` + `lib/schema.ts:32-38` (table exists) — NO frontend form anywhere
- **WHAT**: Full backend exists (API route, validation, DB table `newsletterSubscribers`) but zero frontend integration. No form component, no fetch call, no link to `/api/newsletter` in any component.
- **IMPACT**: Feature is unreachable. Newsletter table will remain empty forever. Dead code across API + schema layers.

#### 2. Admin Content Reads Bypass API Route
- **WHERE**: `app/(admin)/admin/content/page.tsx:96` calls `listContent()` directly; `app/(admin)/admin/content/[key]/page.tsx:41` calls `getContent()` directly. Meanwhile `app/api/content/route.ts:7-14` exports a GET handler.
- **WHAT**: Admin pages call `lib/content.ts` server-side for reads, but use HTTP PUT for writes. The API GET endpoint is dead code — never called.
- **IMPACT**: Any future middleware/logging/caching on the GET route would be silently bypassed. Read and write paths diverge structurally.

#### 3. Session Expiry Silent Failure in Content Editor
- **WHERE**: `app/(admin)/admin/content/[key]/ContentEditor.tsx:454-456` — catches 401 as generic error
- **WHAT**: When session expires mid-edit, PUT returns 401. Editor shows "Unauthorized" as a generic error. No redirect to login, no session-expired message, no recovery path.
- **IMPACT**: User loses context, doesn't understand why saves fail, retries futilely.

### HIGH — Config/Data Silently Dropped

#### 4. `/api/content` Has No Rate Limiting
- **WHERE**: `app/api/content/route.ts` — no `checkRateLimit()` call (other 3 routes all have it)
- **WHAT**: Admin mutation endpoint relies only on session auth. No throughput protection.
- **IMPACT**: Compromised session can hammer PUT requests without throttling.

#### 5. ManifestoSection Hardcoded Attribution (SSR/Hydration Drift)
- **WHERE**: `data/home.ts:34` exports `manifesto.attribution` → `components/home/ManifestoSection.tsx:64` uses it pre-mount, but line 93 hardcodes `"— Jesús Herrera, fundador"` post-mount
- **WHAT**: Mounted render ignores data file, uses hardcoded string.
- **IMPACT**: Changes to `manifesto.attribution` in data only appear during SSR flash, then get replaced on hydration.

#### 6. `blogCategories` + `getCategoryMeta` Dead Exports
- **WHERE**: `data/blog.ts:3-38` — defines 5 categories with colors/descriptions
- **WHAT**: Exported but never imported anywhere. MDX posts define category as raw strings with no link to this registry.
- **IMPACT**: Category metadata (colors, descriptions) is completely disconnected from rendering. Dead code.

#### 7. Blog `featured` Flag Parsed But Never Consumed
- **WHERE**: `lib/blog.ts:14,36,63` extracts `featured: boolean` from frontmatter → no component reads it
- **WHAT**: Both posts define `featured: true/false` but PostGrid, PostCard, PostHeader never reference the field.
- **IMPACT**: No visual distinction for featured posts. Flag is silently ignored.

#### 8. Blog `image` Field Parsed But Never Rendered
- **WHERE**: `lib/blog.ts:17,39,66` extracts optional `image` from frontmatter → no blog component references `post.image`
- **WHAT**: Field survives parsing but has no consumer.
- **IMPACT**: Adding `image` to a post's frontmatter would be silently ignored.

#### 9. `getContentWithFallback()` Never Called
- **WHERE**: `lib/content.ts:14-25` — exported function
- **WHAT**: Designed for graceful DB fallback during static builds, but no caller exists.
- **IMPACT**: Dead code. If DB fails during build, no graceful fallback kicks in because nothing calls this function.

### MEDIUM — Data Shape Mismatches

#### 10. AboutHero Hardcoded Sectors (Ignores Data)
- **WHERE**: `data/about.ts:38` exports `story.sectors` → `components/about/AboutHero.tsx:141-147` hardcodes sector strings
- **WHAT**: Data file provides sectors array, component ignores it and uses static JSX.
- **IMPACT**: Sector changes in data file won't propagate to UI.

#### 11. Client `sector` Field Never Used
- **WHERE**: `data/clients.ts:7-20` defines `sector` on all clients → `components/home/ClientMarquee.tsx:7` only reads `.name`
- **WHAT**: Every client object carries sector metadata that no component consumes.
- **IMPACT**: Unused data field. Sector-based filtering/grouping impossible without wiring it.

#### 12. CaseStudy `testimonial` + `stats` Types Never Populated
- **WHERE**: `types/content.ts:54-63` defines optional `testimonial` and `stats` on CaseStudy → `data/work.ts:364-398` provides neither
- **WHAT**: Type contract allows these fields, CaseStudyNarrative component accepts them, but actual data never provides them.
- **IMPACT**: Dead code paths in CaseStudyNarrative (testimonial and stats rendering blocks never execute).

#### 13. JSON-LD `articleSchema` Missing Image
- **WHERE**: `lib/jsonld.ts:81-111` — takes only headline, description, datePublished, url
- **WHAT**: No image parameter. Google prefers article image in BlogPosting schema for rich snippets.
- **IMPACT**: SEO degradation — blog posts can't communicate hero images to search engines.

### LOW — Minor Inconsistencies

#### 14. Rate Limiter Cleanup Interval Hardcoded
- **WHERE**: `lib/rate-limit.ts:26` — `CLEANUP_INTERVAL = 60s` regardless of `windowMs` parameter
- **WHAT**: If a route passes a longer window (e.g., 1 hour), cleanup may prune valid timestamps.

#### 15. Blog `readingTime` Not Calculated
- **WHERE**: `lib/blog.ts:62` reads from frontmatter, never computes from content

## Systemic Patterns

### Pattern 1: Data Files Ignored by Components (3 instances)
ManifestoSection, AboutHero, and ClientMarquee all have corresponding data exports that they partially or fully ignore, using hardcoded values instead. This is the most common wiring failure in the project.

### Pattern 2: Parsed-But-Unconsumed Blog Fields (3 instances)
`image`, `featured`, and `blogCategories` are all defined/parsed but never reach rendering. The blog pipeline parses more than it displays.

### Pattern 3: Dead Exports in Lib Layer (2 instances)
`getContentWithFallback()` and `GET /api/content` are built but never called. Backend capabilities exist without frontend consumers.

### Pattern 4: No Auth/Rate-Limit Symmetry
3 of 4 API routes have rate limiting. The one without it (`/api/content`) is the only mutation endpoint. Auth protects it, but the rate-limit gap breaks the security symmetry.
